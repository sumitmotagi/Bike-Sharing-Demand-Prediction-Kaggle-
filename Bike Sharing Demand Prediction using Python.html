Bike Sharing Demand Prediction¶

Introduction:¶

The Kaggle "Bike Sharing Demand" competition is focused on predicting the number of bike rentals in a city based on various features like time, weather conditions, and holidays. This project involves building a robust model to accurately forecast demand for a bike-sharing service, which can help optimize fleet management and improve customer satisfaction. In this project, I conducted a thorough exploratory data analysis (EDA) to identify key patterns and relationships between the features and the target variable (demand). EDA helped in uncovering how factors like season, temperature, humidity, and working days influence the demand for bike rentals. After understanding the data, I applied multiple linear regression to model the demand. To ensure the model was reliable and accurate, I carefully addressed and corrected key model assumptions like linearity, multicollinearity, and homoscedasticity. Through this process, I optimized the model and achieved a **Root Mean Squared Logarithmic Error (RMSLE) of 0.356, demonstrating strong predictive performance and capturing the intricate dynamics of bike-sharing demand. This project highlights my ability to:

  • Perform detailed data exploration.
  • Build and fine-tune predictive models.
  • Handle real-world business problems by addressing model limitations effectively.

Step-by-Step Breakdown of the Project:¶

1. Problem Understanding and Goal Definition¶

Objective : The goal was to predict the total number of bike rentals for each hour based on features such as time, weather conditions, and holiday information. Evaluation Metric : Root Mean Squared Logarithmic Error (RMSLE)** was used to measure model performance, with the aim of minimizing this metric.

2 Exploratory Data Analysis (EDA)¶

Loading the Data : The dataset was loaded and its structure (training and testing data) was reviewed. Feature Inspection : Key features such as season, holiday, working day, weather, temp, humidity, and windspeed were examined. Visualization of Continuous Variables: Used scatter plots to explore the relationships between continuous variables like temp, humidity, and windspeed against bike demand. Visualization of Categorical Variables: Used bar charts to analyze how categorical variables such as season, weather, holiday, and working day affect demand.

3. Assumptions Check and Tackling¶

Multicollinearity: Multicollinearity between features was examined. Features were assessed for redundancy, and one feature was excluded if necessary to address multicollinearity. Autocorrelation : Autocorrelation in the residuals was checked. Lag values were created for the dependent variable (bike demand) to address autocorrelation. Log Transformation : The dependent variable was log-normal, so a log transformation was applied to make the variable normally distributed. Data Preprocessing: Categorical variables like season, weather, and holiday were converted into dummy variables for use in the regression model.

4. Model Building (Multiple Linear Regression)¶

Train-Test Split : A custom train-test split approach was used to handle autocorrelation in the dependent variable, ensuring that time series dependencies were considered. Model Selection: Multiple Linear Regression was chosen as the model. Fit the Model : The model was trained using the training data to predict bike demand.

5. RMSLE Calculation¶

RMSLE Calculation: After fitting the model, the Root Mean Squared Logarithmic Error (RMSLE) was calculated to evaluate the model’s performance. Final RMSLE : An RMSLE of 0.356 was achieved, demonstrating strong predictive performance.

6. Conclusions and Insights¶

Key Insights: Findings were summarized, including peak demand hours and seasons and the impact of weather conditions on bike-sharing demand.

#calculating RMSLE
Y_test_e = []
Y_predict_e = []
for i in range(0,len(Y_test)):
    Y_test_e.append(math.exp(Y_test[i]))
    Y_predict_e.append(math.exp(Y_predict[i]))
log_sq_sum = 0.0
for i in range(0,len(Y_test_e)):
    log_a = math.log(Y_test_e[i] + 1)
    log_p = math.log(Y_predict_e[i] + 1)
    log_diff = (log_p - log_a)**2
    log_sq_sum = log_sq_sum + log_diff
    rmsle = math.sqrt(log_sq_sum/len(Y_test))
